Обязательно для всех работ:
  - использование многопоточности;
  - не забыть про функции thread_yield, thread_join;
  - должно быть не менее двух потоков.

   Варианты лабораторных работ.

1.  Задан список файлов, нужно вернуть список всех слов из семи букв из всех файлов, для каждого файла искать в отдельном потоке. 
2.  Задан список массивов, найти во всех них числа которые делятся на 10. При поиске таких чисел в каждом массиве создавать новый поток.
3.  Задан список файлов, при переписывании в другой каталог, каждый переписывается в своём потоке.
4.  Задан список ip-адресов, сканировать заданные ip-адреса.
5.  В цикле выводить на экран содержимое глобального массива целых чисел. Массив  непрерывно заполняется.
6.  Реализовать: поставка товара возможна после того, как оформлен заказ и оплачен.
7.  Задан список арифметических выражений. Вычислить их.
8.  Для функции задан список аргументов. Вычислить значение функции для каждого аргумента.
9.  Реализовать скачивание и сохранение нескольких небольших файлов по ссылкам.
10. Товары одного заказа можно помещать в одну коробку, разных заказов - нельзя, в коробку помещается не более 5-ти товаров. В каждом заказе: наименование товара, время реализации и количество.
    (В главном потоке происходит генерация заказов по приблизительно следующей схеме: сгенерировать заказ, положить в коробки, сгенерировать еще один (или несколько) и т.д.
    Поток блокируется, если в данный момент отсутствует подходящее количество свободных коробок.)
11. Из заданного списка файлов, нужно выбрать короткие (менее 30 символов) и длинные строки (более 60), записать их в разные файлы. 
12. Дана функция. Вычислять значение функции для постоянно меняющегося  аргумента. Выводить на печать зачения функции и аргумента.
13. Задан список слов. Слова без буквы "а" выводить в файл, слова с буквой "а" - на экран.
14. Реализовать: поток родитель создает файл и создает три дочерних потока, каждый из которых осуществляет запись своего идентификатора в файл 10 раз.
15. Читать строки из достаточно большого файла (более 100 строк), редактировать (оставлять в строке 5 слов) и записывать в другой файл, в 10 потоков.
16. Реализовать: перевод целого числа со знаком в его строковый эквивалент прописью. Перевод числа в потоке. Ввод числа происходит до запуска потока, а вывод строки - по его завершению.
17. Задан список файлов. Файлы с именам, соответствующими шаблону, переписываются в другой каталог (каждый в своём потоке), остальные удаляются.
